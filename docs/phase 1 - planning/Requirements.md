# Requirements

This document describes all the requirements that need to be fulfilled by the development

## 1.0

- the system should be a single-user-system, as it should not support accounts
- the user should be able to read, add, update and delete contacts
- the user should be able to read, add, update and delete messages related to a contact
- the user should be ablet to set a time and date to send a message to a contact
- the system should trigger a daily task to send messages
- the system should persist its data into a database 
- the system should detect failed messages and inform the user
- the user should be able to resend failed messages
